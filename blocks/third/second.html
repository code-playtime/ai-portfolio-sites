BlockForge

A block-based rich text editor built on the Web Components standard. Each block is a fully autonomous <custom-element> that manages its own state, rendering, and events. Blocks can be used standalone or orchestrated by the BlockEditor class which adds a toolbar, drag-to-reorder, and collective save.

Installation
npm install @bhaswanth53/block-forge
With a bundler
import BlockEditor from '@bhaswanth53/block-forge';
import '@bhaswanth53/block-forge/css';
Plain <script> tag
<link rel="stylesheet" href="node_modules/@bhaswanth53/block-forge/dist/block-forge.css" />
<script src="node_modules/@bhaswanth53/block-forge/dist/block-forge.umd.js"></script>
Architecture

BlockForge has two layers:

1
Web Component blocks — text-block, code-block
Each is a native custom element extending HTMLElement. They self-render, manage their own editor instances (Quill / CodeMirror), track mode state (preview/edit), and fire block:save / block:delete custom events that bubble up through the DOM.
2
BlockEditor orchestrator — new BlockEditor(selector)
A plain JS class that builds the toolbar shell, mounts a SortableJS drag list, creates blocks dynamically, listens for bubbling events from all child blocks, and exposes onSave / onSaveAll / load / getAll. Entirely optional — blocks work without it.

Lifecycle flow
HTML parsed → connectedCallback() fires
  → _render() builds innerHTML
  → _attachEvents() wires listeners
  → [user clicks Edit] → setMode('edit')
      → _render() rebuilds as edit UI
      → _mountQuill() / _mountCodeMirror() after 20ms
  → [user clicks Save] → _save()
      → snapshot content from editor instance
      → _setStatus('saved')
      → fire block:save CustomEvent (bubbles)
      → call _onSaveCb if set directly on element
  → [BlockEditor hears block:save]
      → calls onSave callback with corrected order
TextBlock <text-block>

Rich text block powered by Quill JS. Renders HTML preview by default; switches to a Quill editor on demand.

Attributes

CodeBlock <code-block>

Code editing block powered by CodeMirror 5 with syntax highlighting via highlight.js. Preview renders the prv-cb highlighted HTML format.

Attributes
AttributeTypeRequiredDescription
block-idnumber | stringrecommendedUnique identifier returned in save callbacks.
languagestringnoDefault language. Falls back to 'javascript'. Must be one of the supported language values.
labelstringnoLabel shown in the block header.
contentstringnoRaw code string. Do not pass HTML here — pass the raw field from your DB record. If the content contains prv-cb, the component automatically extracts the plain code from inside the <code> tag via innerText.
Methods
method onSave(callback) → this

Same signature as TextBlock. Callback receives the full BlockData including language, content (prv-cb HTML), and raw (plain code string).

method setMode(mode)

Snapshots CodeMirror value before switching to preview. In edit mode, mounts CodeMirror on the textarea via CodeMirror.fromTextArea(), restores raw code, and focuses.

There is a 20ms setTimeout before mounting CodeMirror. This is intentional — it allows the DOM to settle after innerHTML is set before CodeMirror measures the textarea.
internal _buildHtmlOutput() → string

Builds the prv-cb HTML from raw code using highlight.js. Falls back to hljs.highlightAuto() if the language isn't recognized. Override this in a subclass if you want a different preview format.

// Output format:
<div class="prv-cb">
  <div class="prv-cb-h"><span>javascript</span></div>
  <pre><code class="hljs language-javascript">...</code></pre>
</div>
internal _extractRaw(content) → string

Converts stored DB content back to a plain code string for loading into CodeMirror. If the string contains prv-cb, it parses it as HTML and reads codeEl.innerText (which automatically decodes HTML entities). Otherwise strips all tags.

_extractRaw(content) {
  if (!content) return '';
  if (content.includes('prv-cb')) {
    const tmp = document.createElement('div');
    tmp.innerHTML = content;
    const el = tmp.querySelector('code');
    return el ? el.innerText : content; // innerText decodes &lt; etc.
  }
  return content.replace(/<[^>]*>/g, '');
}

BlockEditor

Optional orchestrator class. Instantiate it on a container element and it builds a toolbar, a sortable block list, and an add-block strip. All communication with individual blocks happens through the bubbling block:save / block:delete events.

const editor = new BlockEditor(selector: string) 
const editor = new BlockEditor('#editor-container');
Public methods
callback onSave(callback) → this

Fires when any individual block is saved. The block's order is recalculated at the time of save to reflect the current DOM position (including any drag reordering).

editor.onSave(callback: (block: BlockData) => void)
callback onSaveAll(callback) → this

Fires when the Save All toolbar button is clicked. Receives an array of all blocks in current DOM order. Also marks all blocks as 'saved'.

editor.onSaveAll(callback: (blocks: BlockData[]) => void)
method load(blockArray) → this

Stamps web component elements from a JSON array. Sets attributes on each element so connectedCallback picks them up. Uses raw if present, falls back to content.

Block Data Shape

Every callback and getAll() returns objects of this shape:

type BlockData = {
  id: number | string; // block-id attribute value
  type: 'text' | 'code'; // element type
  order: number; // 1-based DOM position
  label: string; // label input value
  language: string | null; // code blocks only; null for text
  content: string; // HTML output — store in DB
  raw: string; // raw content — use for reload
}

Database Integration
// On page load: restore from DB
const rows = await fetch('/api/blocks').then(r => r.json());
editor.load(rows);
// rows should have: { id, type, label, language, content, raw }

// On single block save: upsert by id
editor.onSave(async (block) => {
  await fetch(`/api/blocks/${block.id}`, {
    method:  'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      type:     block.type,
      order:    block.order,
      label:    block.label,
      language: block.language,
      content:  block.content,  // HTML — render on your site
      raw:      block.raw,      // plain — reload into editor
    }),
  });
});

// On save all: bulk update
editor.onSaveAll(async (blocks) => {
  await fetch('/api/blocks/bulk', {
    method:  'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(blocks),
  });
});
Custom IDs are collision-safe. If you load blocks with IDs 42 and 43, idCounter advances to 44. Any new blocks the user adds will start from 44 onwards.
Creating New Block Types

Any new block type is a class that extends HTMLElement and satisfies the block contract. BlockEditor discovers blocks purely by CSS selector — add your tag name to querySelectorAll and it becomes a first-class citizen.

Required contract

Your class must implement these to work with BlockEditor:

MemberTypeRequiredDescription
connectedCallback()lifecycleyesCalled by browser when element is added to DOM. Initialize state and call _render().
setMode(mode)methodyesSwitch between 'preview' and 'edit'. BlockEditor calls this when adding new blocks.
blockDatagetteryesReturn a BlockData object. getAll() reads this from every block.
onSave(callback)methodyes (standalone)Register a per-instance save callback. Return this for chaining.
block:save eventCustomEventyesFire with bubbles:true and detail: BlockData when saved. BlockEditor listens for this.
block:delete eventCustomEventyesFire before removing the element. BlockEditor uses this to refresh count.
.bf-drag-handleDOM elementyesSortableJS needs this selector inside each block for drag-to-reorder to work.
Full walkthrough — ImageBlock

Here is a complete example of a new block type for uploading and displaying images.

// src/js/ImageBlock.js

class ImageBlock extends HTMLElement {

  // ── 1. Observe attributes you care about ──────────────────
  static get observedAttributes() {
    return ['content', 'label'];
    // 'content' will store the image URL or base64
  }

  // ── 2. Initialize state in connectedCallback ──────────────
  connectedCallback() {
    this._mode     = 'preview';  // all blocks start in preview
    this._content  = this.getAttribute('content') || '';
    this._label    = this.getAttribute('label')   || '';
    this._onSaveCb = null;
    this._render();
  }

  // ── 3. Handle reactive attribute changes ──────────────────
  attributeChangedCallback(name, _, val) {
    if (name === 'content') this._content = val;
    if (name === 'label')   this._label   = val;
    if (this.isConnected)   this._render();
  }

  // ── 4. Public onSave registration ─────────────────────────
  onSave(callback) {
    this._onSaveCb = callback;
    return this; // allow chaining
  }

  // ── 5. blockData getter — must return full shape ──────────
  get blockData() {
    return {
      id:       this._resolveId(),
      type:     'image',
      order:    this._getOrder(),
      label:    this._label,
      language: null,          // null for non-code blocks
      content:  this._content, // URL or base64 — same as raw for images
      raw:      this._content,
    };
  }

  // ── 6. setMode — switch preview / edit ────────────────────
  setMode(mode) {
    this._mode = mode;
    this._render();
    // mount any editor logic after render if needed
  }

  // ── 7. _render — build innerHTML for each mode ────────────
  _render() {
    const isEdit = this._mode === 'edit';
    this.innerHTML = `
      <div class="bf-block bf-image-block${isEdit ? ' is-editing' : ''}">
        <div class="bf-block-header">
          <div class="bf-drag-handle">⠿</div>
          <span class="bf-type-badge" style="background:#f0f4ff;color:#3b5bdb">⎙ image</span>
          <input class="bf-label-input" type="text" value="${this._label}" placeholder="Caption…" />
          <div class="bf-block-controls">
            <span class="bf-save-status"></span>
            ${isEdit
              ? `<button class="bf-icon-btn btn-save">💾</button>`
              : `<button class="bf-icon-btn btn-edit">✎</button>`
            }
            <button class="bf-icon-btn btn-delete">🗑</button>
          </div>
        </div>
        <div class="bf-block-body" style="padding:16px">
          ${isEdit
            ? `<input type="url" class="img-url-input"
                style="width:100%;padding:8px;border:1px solid var(--border);border-radius:6px;font-family:var(--mono);font-size:12px"
                placeholder="Paste image URL…"
                value="${this._content}" />`
            : this._content
              ? `<img src="${this._content}" alt="${this._label}"
                   style="max-width:100%;border-radius:6px;display:block" />`
              : `<span class="bf-empty-hint">No image — click edit to add a URL</span>`
          }
        </div>
      </div>`;
    this._attachEvents();
  }

  // ── 8. _attachEvents — wire up controls ───────────────────
  _attachEvents() {
    this.querySelector('.bf-label-input')
      ?.addEventListener('input', e => { this._label = e.target.value; });

    this.querySelector('.btn-edit')
      ?.addEventListener('click', () => this.setMode('edit'));

    this.querySelector('.btn-save')
      ?.addEventListener('click', () => {
        // Snapshot the URL from the input before switching
        const input = this.querySelector('.img-url-input');
        if (input) this._content = input.value;
        this._save();
      });

    this.querySelector('.btn-delete')
      ?.addEventListener('click', () => this._delete());
  }

  // ── 9. _save — snapshot, fire event, call callback ────────
  _save() {
    this._setStatus('saved');
    this.setMode('preview');         // switch to preview after save
    const data = this.blockData;

    if (this._onSaveCb) this._onSaveCb(data);

    // Must bubble so BlockEditor's listener catches it
    this.dispatchEvent(new CustomEvent('block:save', {
      bubbles:  true,
      composed: true,
      detail:   data,
    }));
  }

  // ── 10. _delete — fire event then remove self ─────────────
  _delete() {
    this.dispatchEvent(new CustomEvent('block:delete', {
      bubbles:  true,
      composed: true,
      detail:   { id: this._resolveId() },
    }));
    this.style.opacity    = '0';
    this.style.transition = 'opacity 0.15s';
    setTimeout(() => this.remove(), 150);
  }

  // ── Helpers (copy these into every block type) ────────────
  _setStatus(state) {
    const el = this.querySelector('.bf-save-status');
    if (!el) return;
    el.textContent = state === 'saved' ? 'saved' : 'unsaved';
    el.className = `bf-save-status visible ${state === 'saved' ? 'is-saved' : 'is-changed'}`;
    if (state === 'saved') {
      clearTimeout(el._t);
      el._t = setTimeout(() => el.classList.remove('visible'), 2200);
    }
  }

  _resolveId() {
    const v = this.getAttribute('block-id') || this.getAttribute('id');
    return v && !isNaN(v) ? Number(v) : v;
  }

  _getOrder() {
    // querySelectorAll must include your new tag
    const list = this.closest('.bf-blocks-list') || this.parentElement;
    if (!list) return 1;
    return [...list.querySelectorAll('text-block, code-block, image-block')]
      .indexOf(this) + 1;
  }
}

customElements.define('image-block', ImageBlock);
export default ImageBlock;
