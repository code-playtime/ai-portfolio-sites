BlockForge v0.0.1
BlockForge

A block-based rich text editor built with web components. Each block is a standalone custom element that can be used independently or inside the BlockEditor orchestrator.

Installation
npm install @bhaswanth53/block-forge
Import
import BlockEditor from '@bhaswanth53/block-forge';
import '@bhaswanth53/block-forge/css';
Web Components

Two custom elements are registered globally when BlockForge is imported.

text-block

Rich text block powered by Quill JS. Starts in preview mode showing rendered HTML.

<text-block
  block-id="1"
  label="Introduction"
  content="<h2>Hello</h2><p>World</p>"
></text-block>
AttributeTypeDescription
block-idnumber | stringUnique identifier. Used in save callbacks.
labelstringOptional display label shown in block header.
contentstring (HTML)Initial HTML content to display and edit.
code-block

Code block powered by CodeMirror + highlight.js. Preview shows syntax-highlighted HTML in the prv-cb format.

<code-block
  block-id="2"
  language="javascript"
  label="Snippet"
  content="console.log('hi')"
></code-block>
AttributeTypeDescription
block-idnumber | stringUnique identifier.
languagestringCode language. See supported languages below.
labelstringOptional label shown in header.
contentstringRaw code string (not HTML).
Supported Languages
javascript  typescript  python  html
css  java  cpp  bash  plaintext
Block Modes

Every block has two modes toggled by the buttons in the block header.

ModeTextBlockCodeBlock
previewRenders HTML output (h2, p, ul…)Renders prv-cb highlighted code

// Switch programmatically
block.setMode('edit');
block.setMode('preview');
Standalone Usage
No BlockEditor needed

Components work independently. Register a save callback directly on the element.

const block = document.querySelector('text-block');

block.onSave((data) => {
  console.log(data.id);       // block-id attribute
  console.log(data.type);     // "text"
  console.log(data.order);    // position in parent
  console.log(data.label);    // label input value
  console.log(data.language); // null for text blocks
  console.log(data.content);  // HTML output
  console.log(data.raw);      // raw content
});
You can also listen for the block:save event which bubbles up from every block, letting you attach one listener on a parent container. 
document.addEventListener('block:save', (e) => {
  const block = e.detail; // same shape as onSave callback
});
BlockEditor Orchestrator
Toolbar + drag + saveAll

Wraps components with a toolbar, drag-to-reorder, and collective save.

const editor = new BlockEditor("#editor-container");
onSave(callback)

Fires when a single block's Save button is clicked.

editor.onSave((block) => {
  console.log(block.id);
  console.log(block.type);     // "text" | "code"
  console.log(block.order);    // 1-based DOM position
  console.log(block.label);
  console.log(block.language); // null for text blocks
  console.log(block.content);  // HTML — store in DB
  console.log(block.raw);      // raw — use for reload
});
onSaveAll(callback)

Fires when the Save All toolbar button is clicked.

editor.onSaveAll((blocks) => {
  // blocks → array of same shape as onSave
  console.log(blocks);
});
load(blockArray)

Pre-populates from a JSON array. Pass raw for editor content and content for stored HTML output.

editor.load([
  {
    id:      42,
    type:    "text",
    label:   "Intro",
    raw:     "<h2>Hello</h2>",
  },
  {
    id:       43,
    type:     "code",
    language: "python",
    label:    "Script",
    raw:      "print('hi')",
  },
]);
getAll()

Returns current state of all blocks in DOM order without triggering callbacks.

const snapshot = editor.getAll();
// → [{ id, type, order, label, language, content, raw }]
Block Data Shape
FieldTypeDescription
idnumber | stringFrom block-id attribute or auto-incremented
type"text" | "code"Block type
ordernumber1-based position in current DOM order
labelstringValue of the label input
languagestring | nullLanguage for code blocks, null for text
contentstring (HTML)Final HTML output — store this in DB
rawstringRaw content — pass back via load() to reload editor
Database Integration
// Save to DB on individual block save
editor.onSave(async (block) => {
  await fetch(`/api/blocks/${block.id}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      content:  block.content,  // HTML
      raw:      block.raw,      // raw
      order:    block.order,
      label:    block.label,
      language: block.language,
    }),
  });
});

// Restore from DB on page load
const rows = await fetch('/api/blocks').then(r => r.json());
editor.load(rows); // pass raw field so editor reloads correctly
Custom IDs are safe — if you load blocks with ids 42 and 43, any new blocks added by the user will start at id 44, avoiding collisions.
Extending with New Block Types

Create a new class extending HTMLElement and register it as a custom element. You only need to implement these:

class ImageBlock extends HTMLElement {
  connectedCallback() { /* init */ }
  setMode(mode)       { /* toggle preview/edit */ }
  get blockData()     { /* return { id, type, order, label, content, raw } */ }
  onSave(cb)          { this._onSaveCb = cb; return this; }
}

customElements.define('image-block', ImageBlock);

Then import it in main.js and add image-block to the querySelectorAll in getAll() and load().
