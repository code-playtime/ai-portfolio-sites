Installation

FlexPHP is a modern PHP framework designed for building robust web applications with elegant syntax and powerful features.

Server Requirements

Before installing FlexPHP, ensure your server meets the following requirements:

  • PHP >= 8.1
  • Composer
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Mbstring PHP Extension

Installing FlexPHP

You can install FlexPHP via Composer. Run the following command in your terminal:

php flex migrate

Rolling Back Migrations

Rollback the last batch of migrations:

php flex migrate:rollback

Rollback all migrations:

php flex migrate:reset

Column Types

The schema builder supports various column types:

<?php

$table->bigIncrements('id');
$table->string('name', 100);
$table->text('description');
$table->integer('votes');
$table->decimal('amount', 8, 2);
$table->boolean('confirmed');
$table->date('created_date');
$table->datetime('created_at');
$table->timestamp('updated_at');
$table->json('options');

Tip: Always create a corresponding down method to reverse your migration changes.

language-bash">composer create-project flexphp/flexphp my-app

After installation, navigate to your project directory:

cd my-app

Local Development Server

FlexPHP includes a built-in development server. To start it, run:

php flex serve

Your application will be available at http://localhost:8000

Note: The development server is only for local development. Use a proper web server like Apache or Nginx in production.

Configuration

All configuration files for FlexPHP are stored in the config directory. Each option is documented, so feel free to explore the files.

Environment Configuration

FlexPHP uses environment variables to manage configuration settings. When you install FlexPHP, a .env.example file is included in the root directory. Copy this file to create your own .env file:

cp .env.example .env

Application Key

After creating your .env file, generate an application key:

php flex key:generate

This key is used for encrypting user sessions and other encrypted data.

Environment Variables

Here's an example of common environment variables:

APP_NAME=FlexPHP
APP_ENV=local
APP_KEY=
APP_DEBUG=true
APP_URL=http://localhost

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=flexphp
DB_USERNAME=root
DB_PASSWORD=

Warning: Never commit your .env file to version control. It may contain sensitive credentials.

Directory Structure

The default FlexPHP application structure is intended to provide a great starting point for both large and small applications.

The Root Directory

The root directory contains several folders and files:

The App Directory

The app directory contains the core code of your application. It includes:

  • Controllers - HTTP controllers
  • Models - Eloquent models
  • Middleware - HTTP middleware
  • Providers - Service providers

The Config Directory

The config directory contains all of your application's configuration files.

The Database Directory

The database directory contains database migrations, factories, and seeds.

The Public Directory

The public directory contains the front controller and your assets (images, JavaScript, CSS, etc.).

The Routes Directory

The routes directory contains all route definitions for your application.

Request Lifecycle

Understanding the request lifecycle helps you comprehend how FlexPHP processes incoming requests.

Lifecycle Overview

When a request enters your FlexPHP application, it goes through the following steps:

  1. The request enters through public/index.php
  2. The HTTP Kernel handles the request
  3. The request passes through middleware
  4. The router dispatches the request to a controller or closure
  5. The response is generated and sent back through middleware
  6. The response is sent to the browser

Entry Point

All requests to your application are directed to public/index.php:

<?php

require __DIR__.'/../vendor/autoload.php';

$app = require_once __DIR__.'/../bootstrap/app.php';

$kernel = $app->make(Kernel::class);

$response = $kernel->handle(
    $request = Request::capture()
);

$response->send();

$kernel->terminate($request, $response);

Service Container

The FlexPHP service container is a powerful tool for managing class dependencies and performing dependency injection.

Binding

You can bind a class or interface to the container using the bind method:

<?php

$this->app->bind(UserRepository::class, function ($app) {
    return new UserRepository($app->make('db'));
});

Resolving

You can resolve a class from the container using the make method:

<?php

$repository = $app->make(UserRepository::class);

Automatic Injection

The container automatically resolves dependencies in controller constructors:

<?php

namespace App\Controllers;

use App\Repositories\UserRepository;

class UserController extends Controller
{
    protected $users;

    public function __construct(UserRepository $users)
    {
        $this->users = $users;
    }
}

Service Providers

Service providers are the central place to configure your application. They bind services into the service container.

Writing Service Providers

All service providers extend the ServiceProvider class:

<?php

namespace App\Providers;

use FlexPHP\Support\ServiceProvider;

class AppServiceProvider extends ServiceProvider
{
    public function register()
    {
        $this->app->bind('custom', function () {
            return new CustomClass();
        });
    }

    public function boot()
    {
        // Bootstrap application services
    }
}

Registering Providers

Register your service providers in config/app.php:

<?php

'providers' => [
    App\Providers\AppServiceProvider::class,
    App\Providers\RouteServiceProvider::class,
],

Routing

FlexPHP routes are defined in the routes directory. The framework automatically loads these files.

Basic Routing

The most basic FlexPHP routes accept a URI and a closure:

<?php

Route::get('/welcome', function () {
    return 'Welcome to FlexPHP!';
});

Route Parameters

You can capture segments of the URI within your route:

<?php

Route::get('/user/{id}', function ($id) {
    return 'User '.$id;
});

Named Routes

Named routes allow you to conveniently generate URLs or redirects:

<?php

Route::get('/profile', function () {
    // ...
})->name('profile');

// Generating URLs
$url = route('profile');

Route Groups

Route groups allow you to share attributes across multiple routes:

<?php

Route::prefix('admin')->group(function () {
    Route::get('/users', function () {
        // Matches /admin/users
    });
});

Middleware

Middleware provides a convenient mechanism for filtering HTTP requests entering your application.

Defining Middleware

Create a new middleware using the make command:

php flex make:middleware CheckAge

This command will create a new middleware class:

<?php

namespace App\Middleware;

use Closure;

class CheckAge
{
    public function handle($request, Closure $next)
    {
        if ($request->age <= 18) {
            return redirect('home');
        }

        return $next($request);
    }
}

Registering Middleware

Assign middleware to routes in your route files:

<?php

Route::get('/admin', function () {
    // ...
})->middleware(CheckAge::class);

Controllers

Controllers group related request handling logic into a single class, stored in the app/Controllers directory.

Basic Controllers

Here's an example of a basic controller:

<?php

namespace App\Controllers;

use App\Models\User;

class UserController extends Controller
{
    public function index()
    {
        $users = User::all();
        return view('users.index', ['users' => $users]);
    }

    public function show($id)
    {
        $user = User::find($id);
        return view('users.show', ['user' => $user]);
    }
}

Resource Controllers

FlexPHP resource controllers provide CRUD operations:

<?php

Route::resource('photos', PhotoController::class);

This creates multiple routes for typical CRUD operations:

  • GET /photos - index
  • GET /photos/create - create
  • POST /photos - store
  • GET /photos/{id} - show
  • GET /photos/{id}/edit - edit
  • PUT/PATCH /photos/{id} - update
  • DELETE /photos/{id} - destroy

Requests

FlexPHP's request class provides an object-oriented way to interact with the current HTTP request.

Accessing the Request

Obtain an instance of the current request via dependency injection:

<?php

namespace App\Controllers;

use FlexPHP\Http\Request;

class UserController extends Controller
{
    public function store(Request $request)
    {
        $name = $request->input('name');
        // ...
    }
}

Retrieving Input

Access all input data using the all method:

<?php

$input = $request->all();

Retrieve a specific input value:

<?php

$name = $request->input('name');
$email = $request->input('email', 'default@example.com');

Request Validation

Validate incoming request data:

<?php

$validated = $request->validate([
    'name' => 'required|max:255',
    'email' => 'required|email',
    'age' => 'required|integer|min:18',
]);

Responses

All routes and controllers should return a response to be sent back to the user's browser.

Basic Responses

Return strings or arrays from routes:

<?php

Route::get('/', function () {
    return 'Hello World';
});

Route::get('/json', function () {
    return ['name' => 'FlexPHP', 'version' => '1.0'];
});

Response Objects

Create response instances for more control:

<?php

return response('Hello World', 200)
    ->header('Content-Type', 'text/plain');

Redirects

Redirect responses are instances of the RedirectResponse class:

<?php

return redirect('/home');

return redirect()->route('profile');

return redirect()->back();

JSON Responses

The json method automatically sets the Content-Type header:

<?php

return response()->json([
    'name' => 'John',
    'status' => 'active'
]);

Database Setup

FlexPHP makes interacting with databases extremely simple across a variety of database backends.

Configuration

Database configuration is located in config/database.php. You can configure multiple database connections:

<?php

'connections' => [
    'mysql' => [
        'driver' => 'mysql',
        'host' => env('DB_HOST', '127.0.0.1'),
        'port' => env('DB_PORT', '3306'),
        'database' => env('DB_DATABASE', 'flexphp'),
        'username' => env('DB_USERNAME', 'root'),
        'password' => env('DB_PASSWORD', ''),
    ],
],

Running Queries

You can run queries using the DB facade:

<?php

use FlexPHP\Support\Facades\DB;

$users = DB::select('SELECT * FROM users WHERE active = ?', [1]);

Query Builder

FlexPHP's query builder provides a convenient, fluent interface for creating and running database queries.

Retrieving Results

Retrieve all rows from a table:

<?php

use FlexPHP\Support\Facades\DB;

$users = DB::table('users')->get();

Retrieve a single row:

<?php

$user = DB::table('users')->where('name', 'John')->first();

Where Clauses

Add where clauses to your queries:

<?php

$users = DB::table('users')
    ->where('votes', '>', 100)
    ->where('status', 'active')
    ->get();

Ordering and Grouping

Order and group query results:

<?php

$users = DB::table('users')
    ->orderBy('name', 'desc')
    ->groupBy('account_id')
    ->get();

Inserts

Insert records into the database:

<?php

DB::table('users')->insert([
    'email' => 'john@example.com',
    'name' => 'John Doe'
]);

Updates

Update existing records:

<?php

DB::table('users')
    ->where('id', 1)
    ->update(['votes' => 1]);

Deletes

Delete records from the database:

<?php

DB::table('users')->where('votes', '<', 100)->delete();

Migrations

Migrations are like version control for your database, allowing your team to define and share the application's database schema.

Generating Migrations

Create a new migration using the make command:

php flex make:migration create_users_table

Migration Structure

A migration class contains two methods: up and down:

<?php

use FlexPHP\Database\Migrations\Migration;
use FlexPHP\Database\Schema\Blueprint;
use FlexPHP\Support\Facades\Schema;

class CreateUsersTable extends Migration
{
    public function up()
    {
        Schema::create('users', function (Blueprint $table) {
            $table->id();
            $table->string('name');
            $table->string('email')->unique();
            $table->timestamp('email_verified_at')->nullable();
            $table->string('password');
            $table->rememberToken();
            $table->timestamps();
        });
    }

    public function down()
    {
        Schema::dropIfExists('users');
    }
}
